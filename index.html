Overview

The Discord chat platform and RMIT website will be the core of this project. This will be a FAQ or Chat Bot with the Speech Recognition feature to allow a user to request RMIT information through a voice channel and to produce a response using an online API to parse/interpret natural language returned either as a visual or or verbal chat response. While there are a wide variety of uses for this technology, the primary focus of this project will be for visually impaired people who have issues understanding the content in the RMIT and RMIT.edu websites. As it stands, most of the information is displayed as text and videos are difficult to decipher from the dialogue. This project aims to create an interactive medium to provide some of the functionality found in a mobile application while providing accessibility and use with words for the visually impaired. The project is targeted mostly to visually disabled participants and, if they so wish, will be accessed by anyone.

Motivation

Delivering audio content is a major problem when it comes to accessibility. It’s hard to understand what someone is trying to say when the audio just sounds like noise and very few platforms provide that with the audio in a format that’s suitable for people with hearing or visual disabilities. This project would be fascinating, as developers/students would be introduced to the processing of artificial intelligence/natural language as well as to programming to help improve skills for potential projects and/or careers. This project is believed to be significantly beneficial, as visually disabled adults could ask questions orally and have an audio format returned to them. I already have familiarity with Discord chat bots and the means to host these bots, which is very open and affordable either by cloud hosts such as Digital Ocean, Amazon Web services or a single-board machine such as a Raspberry Pi.

Description

The project will be managed on the Discord chat platform through the Discord Developer API and any APIs for natural language processing, like wit.ai or api.ai. Across Node.JS, Python, Java, NET / C++ and more the bot can run in the required language in the Discord Bot Library chosen for use. The other key focus of the Bot Library is that it integrates with many bot frameworks in order to facilitate the building of bots. One of the major issues facing bot makers today is trying to curate and make sense of all the chatbots that are out there. Being able to build bots that are native to Discord is going to greatly increase developer flexibility. In addition, the Bot would be hosted on either Linux device, on a Digital Ocean or Amazon Web Services Virtual Private Server or on the single-board machine such as a Raspberry Pi and other related items. The bot will use a Voice Channel on the Discord platform and wait either always on or on the use of a trigger word for verbal user input. When verbal user feedback is obtained, the query is forwarded to the natural language processor API. The web scraper aims to collect the information directly from RMIT's website by retrieval of the corresponding information, which will be converted to the bot back into a text to the speech generator delivering a hearing response, by transmitting one of two units, one a web scraper and one FAQ. Otherwise the chat bot will respond to the user using the data stored for a particular entry, using the same text for the speech answer mechanisms as the web-scraping process, when the query fits the question stored in the FAQ module. The bot will also return different responses to questions which require more processing time such as when passing a question from the FAQ module to the web-scraper. This is a work in progress, and a limitation will be that a user’s question will not be logged unless they engage in an actual conversation with the bot. But the bot will have an option to store, print and create a digital profile upon interaction. Although the limitations to this project are the difficulty with the audio stream that is already being transmitted to the bot and the likelihood of multiple audio streams interacting with it when the bot is already used in more than one place/server).). There is also a substantial investment time required to set up the natural language processor to translate precisely what is the user demand and thus to have the right information. In contrast, a traditional bot which is simply programmed to understand a set of keywords and automatically respond by providing the information requested is much faster and is capable of being used on any device (with the exception of small Android screens, which require way too many taps). The above details would be of interest to any developer that is developing a bot capable of conversing with people. Chat platforms like Discord.net are always looking for new ways to improve their system. At present, Discord.net is the only platform to have provided integrated, real-time bot integrations with several vendors such as Google Home and Amazon Echo devices.

Tools and Technologies

Any Linux computer, such as a Raspberry-Pi or other similar devices, will be needed for this project hardware. Or a virtual private server ( VPS) from a vendor that enables a developer to operate remotely such as Digital Ocean. Amazon's Web Services. Software includes the use of the API for natural language processing, such as wit.ai, api.ai and a similar discord bot library and language, such as Discord.JS (node.js), JDA (Java), Discord.NET (.NET Core) (Python). Ffmpeg, which would allow a bot to receive audio from the chat platform, would also be other resources needed.

Skills Required

This project requires several skills as below: ● Linux system experience ● Programing familiarity of Disk Bot Libraries in particular ● API working experience ● Experience with functions associated with ffmpeg/audio ● Familiarity with website manipulation to collect data via API calls or web scraping Any software that needs to be written in the language in question will be picked from the bot repository. Such hardware is not needed since it is necessary to use either a Digital Ocean server or the Web Services Amazon Server, but it can also be used with a Raspberry Pi or with some other related device on Linux. The probability to find the required software and hardware is very strong since it is open source software while the hardware, be it a VPS, or a Raspberry Pi, is easily accessible. As regard to skills however the software is the more relevant – although in practice, if the components are already known or the requirement is to find a particular piece of software then such as task would be possible. The full source code for Python Bot using the Node.js framework is also available, although to use this programming language one would need to know Python at least somewhat.

Outcome

If this project succeeds, it results in a working chatbot, which could verbally or visually acknowledge the user's feedback and deliver the required response to the user. It will encourage visually disabled users who simply don't want to open a web browser to access data from the RMIT website easily and conveniently from inside the Discord chat app. Furthermore this project may also be built to avoid the use of a chat network and to be implemented separately in a kiosk, for instance.
